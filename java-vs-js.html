Java vs JavaScript (JS)
Differences Similarities
Java is object-oriented and class-based, while JS is a scripting language Java is compiled, JS is interpreted
Java is better used for back-end development, while JS is better fro front-end Can be used for front-end/client-side & back-end/server-side web development
Java is standalone, while JS is not Both have a lot of 3rd-party support through frameworks, plug-ins, libraries, etc
Java uses more RAM. JS is lightweight Both can run in a browser, although Java applets are becoming less popular
Java supports multi-threading, while JS does not Both can have object classes, and their relationships, such as inheritance
The file extensions are different: .java | .js Both are high-level programming languages
Java variables are type specific and checked at compile time, while JS variables can change types because they are checked dynamically The syntax for most commonly used things, like loops, are the same
for example: while(condition) {code;}
Method/function signatures in Java are more complex, requiring visibility keyword and return type, while JS is simply
function myFunction(params) {code;}		 Both have their own security vulnerabilities that the developer must guard against, for example: SQL injection for Java and Code injection for JS
Java can be used to create any kind of application or program, while JS is primarily a web technology Developers of either language are generally well paid
Java runs in a JVM (Java Virtual Machine), while JS runs primarily in a web browser Both are cross-platform
