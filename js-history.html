JavaScript (JS) History Table
Year Event Notes
1995 JS initial release JS had two beta releases in its first year, both on the Netscape Navigator browser. Its creator was Brendan Eich. The original JS code name was "Mocha".1
1997 ECMA-262 standardization of JS specification ECMAScript (ES) is the primary standard version of JS, while ISO/IEC is secondary. JS is the language, ES is the official version.1
1998 ES version 2 release This update kept the ECMA and ISO standards in sync.1
1999 ES version 3 release Many core features added: regex, control statements, better string and exception handling, etc.1
2005 Ajax formally introduced Jesse James Garrett introduced the now-popular JS tool, Ajax, which was founded in 1999, causing JS to be recognized as a professional programming language. Ajax synchronizes JS and HTML using XML or JSON.2
2006 jQuery initial version release JQuery is a JS library which makes it easier to navigate the HTML DOM.2
2008 ES version 4...doesn't release? This was supposed to be a massive update, but turned out to be too big. This was listed as abandoned.1
2009 ES version 5 release Some standard features and strict mode added.1
2009 NodeJS introduced by Ryan Dahl NodeJS is a very popular server-side and cross-platform JS runtime environment.2
2011 ES version 5.1 release Small update to synchronize ECMA adn ISO standards.1
2015 ES version 6 release Large update bringing many features that were supposed to be included in version 4. Officially named "ECMAScript 2015", signifying the start of yearly update releases. New, "One JavaScript" approach dictates that future versions are always backward compatible.1
2016 ES2016 yearly release Fewer new features added with more frequent updates, but now features ready earlier don't have to wait as long to be adopted.1
2016 AngularJS released Angular is a very popular JS framework.2
2017 ES2017 release Subsequent release timing is standardized to June.1

This table lists a lot of the highlights in JavaScript's development over the last 26 years. There are a lot of other JS tools, frameworks, and libraries out there, such as the popular ReactJS. One other thing to point out is that the modern JS developer no longer needs to worry about whether a site supports a current ECMA version of JS, but whether or not an individual feature they want to use has reached stage 4 in the standardization process, as of 2016. Current progress on proposed features can be reviewed on GitHub.1
